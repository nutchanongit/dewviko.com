งาน: สั่ง AI วิจารณ์ Product

สิ่งที่เขาสร้างเตรียม

ไว้ก่อนที่จะสั่งให้ AI

ทำงานนี้ได้เก่งประกอบด้วย 3 ส่วน

ไฟล์รายละเอียด Product Strategy (งานที่จะให้ai เข้ามาวิเคราะห์)

เขาเขียนเป็น markdown file เก็บไว้ในโฟลเดอร์ projects

เช่น my_product.md

ใน ไฟล์นี้จะมีหัวข้อที่เล่าเกี่ยวกับ รายละเอียดโปรดักษ์ ที่เขาทำ ประมาณนี้คับ

  • Product ทำอะไร
  • Problem ที่จะแก้
  • Target Audience
  • Value Proposition
  • Business Model
  • Strategic Differentiation

knowledge

เป็นFolder เก็บ Best Practice Framework

ที่เขาใช้ในการวาง Strategy โปรดักษ์จากประสบการณ์เขา เช่น

  • what-great-strategy-looks-like.md
  • great-value-proposition.md
  • great-target-audience.md
  • great-strategic-differentiation.md

skill.md

ไฟล์ข้อความที่บอก AI ว่า งานนี้ทำยังไง จะมีลักษณะประมาณนี้ครับ

แบ่งเป็น 3 ส่วน

| 1. Front Matter (ป้ายชื่อ)

| ---

| name: ชื่อ skill

| description: ทำอะไร

| ---

| 2. Goal (เป้าหมาย)

| บอกว่างานนี้ต้องการอะไร โทนแบบไหน

|

| 3. Workflow (ขั้นตอน)

| Step 1 ทำอะไร ...

| Step 2 ทำอะไร ...

| Step 3 ทำอะไร ...

| Step 4 บันทึกผลลัพธ์

ภาพรวมโฟลเดอร์

เขาใช้ VS CODE ในการโชว์ไฟล์โฟลว์เดอร์ที่เขาวางระบบไว้ทั้งหมด

เพื่อให้เห็นภาพรวมของไฟล์ได้ง่ายๆ

จากนั้นเปิด Command Line เพื่อสั่งงาน

ภาพรวมโฟลเดอร์เป็นประมาณนี้ครับ

product-hub/ ← โฟลเดอร์หลักของโปรเจกต์

│

├── .claude/skills/ ← เก็บ skill ทั้งหมด

│ └── critique-product-strategy/ ←skill วิจารณ์

│ ├── skill.md

│ └── knowledge/

│ ├── what-great-strategy-looks-like.md

│ ├── great-value-proposition.md

│ ├── great-target-audience.md

│ ├── great-differentiation.md

│ ├── great-business-model.md

│ └── great-go-to-market.md

│

└── projects/

└── my_product.md ← ตัวที่จะโดนวิจารณ์

การสั่ง Claude Code ทำงานนี้

พอทุกอย่างพร้อมแล้ว เขาสั่ง Claude Code ทำงานแบบนี้ครับ

/skill critique-product-strategy @projects/my_product.md

แปลเป็นภาษาคนคือ จงใช้ Skill วิจารณ์กลยุทธ์ของโปรดักษ์ เพื่อวิจารณ์โปรดักษ์ my_product

แบบนี้ AI จะทำงานได้ตรงตาม Best Practice ของเขา

ไม่มั่ว ไม่ตอบกลางๆ เหมือน มี Specialist มาทำงานให้เลย
